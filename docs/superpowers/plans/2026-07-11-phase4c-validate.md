# Phase 4c — port the `validate` LLM-audit command to the Rust CLI

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use `- [ ]`.

**Goal:** A pure-Rust `validate` subcommand in `nameparser-cli` that mirrors the Java CLI's `validate` — streams a name corpus through the parser, samples a bounded, seeded "suspicious tail" + baseline, sends batches to an LLM (Anthropic / OpenAI-compatible / local Ollama) that judges whether each `ParsedName` faithfully represents the raw input, caches verdicts, and writes a JSONL report of flagged rows for human review.

**Architecture:** A new `crates/nameparser-cli/src/validate.rs` module (the subcommand is the biggest, so keep it out of the already-large `main.rs`) with a `Validate(ValidateArgs)` clap variant dispatched from `main.rs`. Reuses the core `nameparser::parse` + the model's proven `Serialize` (the `parsed` payload/report field is byte-identical to Java's Gson output — no new serialization). New deps, **CLI-crate only** (the core stays lean): `ureq` (blocking HTTP, no async runtime), `sha2` (cache key). Everything else — the `java.util.Random` LCG, reservoir sampling, retry/backoff, the tolerant JSON salvage — is hand-ported (no `rand`/`reqwest`/`tokio`).

**Tech Stack:** Rust 2021, `clap`, `ureq`, `sha2`, `serde`/`serde_json` (reused), `regex` (reused). LLM providers over HTTP; no vendor SDK (Java uses none either).

**THE BLUEPRINT — read before starting:** `docs/superpowers/findings/2026-07-11-validate-java-recon.md` — a full, verified map of the Java subsystem (every option, the exact predicates/regexes, the full prompt text, the request/response shapes, the cache format, the report shape, and per-decision Rust-port notes). Java source: `/Users/markus/code/gbif/name-parser/name-parser-cli/src/main/java/org/gbif/nameparser/cli/` (`ValidateCli.java`, `Reservoir.java`, `BarcodeOtuFilter.java`, `llm/*.java`) + `VALIDATE.md` + the tests in `src/test/`. **Port behaviour faithfully; the recon + Java source are the source of truth for verbatim details (prompt, patterns, request bodies).**

## Global Constraints (decisions — do not re-litigate)

- **New deps go ONLY in `crates/nameparser-cli/Cargo.toml`** (`ureq`, `sha2`); the core `nameparser` crate stays dependency-lean. Hand-roll the `java.util.Random` LCG, `Reservoir`, retry/backoff (no `rand`, no `reqwest`, no `tokio`).
- **Input: plain-text only** — name = substring before the first TAB, trimmed; skip blank/`#` lines. This matches the existing `parse`/`benchmark` readers. **ColDP TSV/CSV auto-detection is explicitly deferred** (the Rust `parse` command doesn't do it either); document it as a v1 limitation.
- **Omit the optional `canonical` field** from the prompt payload — the Rust core has no `NameFormatter` yet (a separate deferred Phase-1 item), and Java treats `canonical` as best-effort (try/catch, silently omitted). Document the deferral; everything else in the payload is present.
- **Reproducible sampling:** hand-roll `JavaRandom` (the 48-bit LCG of `java.util.Random`: `seed = (seed ^ 0x5DEECE66D) & ((1<<48)-1)` on set; `next(bits)` advances `seed = (seed*0x5DEECE66D + 0xB) & mask`, returns `(seed >>> (48-bits))`; `nextDouble()` = `((next(26)<<27) + next(27)) / (1<<53) as f64`). This gives same-`--seed`-same-sample reproducibility AND bit-identical selection vs the Java tool (a bonus — verify against known `java.util.Random` sequences in a test).
- **Port the prompt VERBATIM** (`ValidationPrompt`): the SYSTEM text and the OUTPUT_INSTRUCTION exactly as in the recon/Java (`VERSION = "v1"`, baked into the cache key). Do not paraphrase the parser conventions.
- **Providers:** `anthropic` (default model `claude-opus-4-8`, `x-api-key` from `ANTHROPIC_API_KEY` else `ANTHROPIC_AUTH_TOKEN` bearer, `/v1/messages`, JSON-schema-constrained output) and `openai`/`local`/`ollama` (default model `qwen2.5:14b-instruct`, `OPENAI_API_KEY` optional bearer, `/v1/chat/completions`, `response_format:{type:"json_object"}`, base `http://localhost:11434`). `local`/`ollama` normalize to the openai client. Match the request bodies + `max_tokens = min(32000, 2000 + batch*400)` from the recon.
- **Permissive verdict typing** (match Java): a verdict string that isn't case-insensitively `wrong`/`suspect` buckets as `ok`. Tolerant response parsing (think-tag strip, string-aware brace-matching salvage of a truncated batch) — hand-port with the Java client tests essentially verbatim.
- **No live API in tests/CI.** Unit-test the pure pieces (filter, predicate, JavaRandom, reservoir, cache, prompt payload, response parsing via recorded fixtures); the end-to-end test uses `--dry-run --cache=none` (reproducible, no API calls). A live smoke against local Ollama is optional/manual.
- SPDX headers; crate `0.0.0`. **Working dir** `/Users/markus/code/gbif/name-parser-rust/`. Preamble: `export PATH="$HOME/.cargo/bin:$PATH"`.

## Task 1: `validate` scaffold + args + `BarcodeOtuFilter` + `is_interesting`

**Files:** Create `crates/nameparser-cli/src/validate.rs`; modify `crates/nameparser-cli/src/main.rs` (add the `Validate` variant + dispatch + `mod validate;`), `crates/nameparser-cli/Cargo.toml` (add `regex` if not present; `ureq`/`sha2` can be added in the tasks that first need them, or all here).

- [ ] **Args:** a `ValidateArgs` clap struct with every option from the recon's table (`--provider` default `anthropic`; `--model` (default resolved per provider); `--input` (default `data/col-names.tsv` — but since we only ship plain-text, document); `--output` default `validate-report.jsonl`; `--budget` 2000; `--sample-normal` 200 (clamped `min(sample_normal, budget)`); `--batch` 25 (clamped `max(1, batch)`); `--seed` 17; `--cache` default `validate-cache.jsonl` (literal `none` disables persistence); `--api-url`; `--dry-run`). Wire `Command::Validate(a) => run_validate(a)` (stub `run_validate` returning `Ok(())` for now).
- [ ] **`BarcodeOtuFilter`:** two `LazyLock<Regex>` (mirror Java verbatim): UNITE_SH `(?i)^SH\d{5,}(\.\d+)?FU\b`, BOLD_BIN `(?i)^BOLD:[A-Z]{2,5}\d+\b`. `fn is_barcode_otu(name: &str) -> bool` = strip then either regex `.is_match` at start (anchored `^`, `find`-semantics). Unit-test the exact positives/negatives from `BarcodeOtuFilterTest` (`SH1957732.10FU`, `sh…fu`, `BOLD:AAA0001`, `bold:aab5053` → true; `Abies alba Mill.`, `Shorea`, `Boldenaria`, ``, and a null-equivalent → false).
- [ ] **`is_interesting`:** define a small `ParseOutcome` = `Result<ParsedName, ParseError>` (or reuse `nameparser::parse`'s return). `fn is_interesting(outcome) -> bool`: true if `Err`; else on the `ParsedName` true if `!warnings.is_empty()` OR `state != State::Complete` OR `type_ != NameType::Scientific`. Unit-test each branch (an `Err`; a name with a warning; a `PARTIAL` state; an `OTHER`/`FORMULA` type; a clean `SCIENTIFIC` `COMPLETE` → false).
- [ ] Build (`cargo build -p nameparser-cli`) + `cargo test -p nameparser-cli` green. Commit `Phase 4c: validate scaffold + args + barcode/OTU filter + is_interesting`.

## Task 2: `JavaRandom` + `Reservoir` + the select scan + `--dry-run` selection

**Files:** `crates/nameparser-cli/src/validate.rs` (extend).

- [ ] **`JavaRandom`** — the 48-bit LCG per the Global Constraints. Methods `new(seed: i64)`, `next(bits: u32) -> i32`, `next_double() -> f64`. Test against known `java.util.Random` output for a fixed seed (e.g. `new(17)` then a few `next_double()` — capture the reference values by running `jshell`/a tiny Java snippet, or use published test vectors; assert bit-identical).
- [ ] **`Reservoir<T>`** — Algorithm R using `JavaRandom`: `new(capacity, seed)`, `offer(item)` (`seen++`; if `capacity==0` return; if `len < capacity` push; else `j = (next_double()*seen) as i64`, if `j < capacity` replace `items[j]`), `into_items()`. Test: same seed ⇒ identical sample; capacity 0 ⇒ empty; capacity ≥ N ⇒ all.
- [ ] **`select(...)`** — single pass over the input (plain-text reader, name = pre-first-TAB trimmed, skip blank/`#`), tracking `line`. Per name: if `is_barcode_otu` → skip (count excluded); else `parse` → an item `{line, input, outcome}`; classify via `is_interesting` → offer to `interesting` reservoir (`cap = max(0, budget - sample_normal)`, seed `seed`) else `ordinary` reservoir (`cap = sample_normal`, seed `seed+1`). Every 500_000 rows, `eprintln!("  scanned {n}…")`. After: `chosen = interesting.into_items() ++ ordinary.into_items()`, **sorted by `line` ascending**. Return `chosen` + scan counts. Print a scan summary to stderr.
- [ ] **`--dry-run` (selection half):** run `select`, chunk into `--batch` batches, print `"Dry run: built {b} batches for {n} names, no API calls made. Report → {path}"` (the first-batch payload dump comes in Task 3, once the prompt exists — leave a TODO). Write the verdict-less report rows (line/input/parsed-or-error, no verdict) to `--output`.
- [ ] Test: a `--dry-run --cache=none` over a small committed corpus is **byte-identical across two runs** with the same `--seed` (reproducibility — mirror `ValidateCliTest.dryRunIsReproducible`); barcode/OTU rows excluded; interesting-vs-ordinary split sane. Commit `Phase 4c: JavaRandom + Reservoir + select scan + dry-run selection`.

## Task 3: `ValidationPrompt` + verdict model + tolerant response parsing

**Files:** `crates/nameparser-cli/src/validate.rs` (or a `validate/llm.rs` submodule).

- [ ] **`ValidationPrompt`:** `VERSION = "v1"`; the SYSTEM and OUTPUT_INSTRUCTION **verbatim** from the recon/Java. `user_message(batch: &[Item]) -> String` = `"Judge each of the following {n} parser results.\n" + <compact JSON array>`, each item `{index, input, parsed?: <ParsedName serde tree, nulls omitted>, unparsable?: {type, message}}` (OMIT `canonical` — deferred). Reuse the model's `Serialize` for `parsed`.
- [ ] **Verdict model:** `Verdict { index: usize, verdict: String, confidence: String, fields: Vec<FieldIssue>, note: Option<String> }`, `FieldIssue { name, parsed, expected, reason: String }` (all coerced to display strings — port Java's coercion for non-string field values). `is_ok()` = `verdict.eq_ignore_ascii_case("ok")`.
- [ ] **Tolerant `parse_verdicts(reply: &str) -> Result<Vec<Verdict>>`:** strip `<think(ing)>…</think(ing)>` (even with embedded braces); locate `"verdicts"` then `[`; a string-literal-and-escape-aware, brace-depth element walk that salvages complete objects and drops a trailing unbalanced one (max_tokens cutoff); error if blank or no `"verdicts"` key. **Port the Java client tests essentially verbatim** (`AnthropicClientTest`/`OpenAiClientTest`): think-tag stripping, markdown-fence/preamble tolerance, truncated-batch salvage, nested-object field coercion.
- [ ] Wire the Task-2 `--dry-run` first-batch payload dump to `ValidationPrompt::user_message` of the first `min(batch, chosen.len())` items → stderr. Commit `Phase 4c: validation prompt + verdict model + tolerant JSON salvage`.

## Task 4: `VerdictCache` + LLM clients (`ureq`: anthropic + openai/local) + retry

**Files:** `crates/nameparser-cli/src/validate.rs` (or `validate/{cache,client}.rs`); `Cargo.toml` (add `ureq`, `sha2`).

- [ ] **`VerdictCache`:** JSONL `{"key":"<sha256-hex>","verdict":{…}}`. `open(path)` loads an existing file into a `HashMap`, opens append writer (create+append); `none` mode = in-memory only (per the recon: `put` updates the map even when disabled). `cache_key(version, model, input, shape) -> hex` = SHA-256 over each part's UTF-8 bytes with a `\0` separator after EVERY part (`shape` = the `parsed`/`error` serde JSON, or `""`). `get`/`put` (flush after each `put`). Unit-test key stability + a round-trip.
- [ ] **`Judge` trait:** `fn judge(&self, user_message: &str, batch_size: usize) -> Result<Vec<Verdict>>`. **`AnthropicClient`** (ureq): POST `<base>/v1/messages`, headers `x-api-key`/`authorization: Bearer …`+`anthropic-beta`, `anthropic-version: 2023-06-01`; body `{model, max_tokens: min(32000,2000+batch*400), thinking:{type:"adaptive"}, system, messages:[{role:"user",content}], output_config:{format:{type:"json_schema",schema:<verdictSchema()>}}}` (transcribe the schema from Java via `serde_json::json!`); response = concat `content[].text` where `type=="text"` → `parse_verdicts`. **`OpenAiClient`** (ureq, also local/ollama): POST `<base>/v1/chat/completions`, optional `authorization: Bearer` (only if key present); body `{model, temperature:0, max_tokens, stream:false, response_format:{type:"json_object"}, messages:[{role:"system",content: SYSTEM+"\n\n"+OUTPUT_INSTRUCTION},{role:"user",content}]}`; response = `choices[0].message.content` → `parse_verdicts` (degrade to `vec![]`+warn on parse failure, like Java). **Retry** (both): `MAX_ATTEMPTS=4`, on HTTP 429/≥500 back off (`retry-after` if present else `2^attempt * 500ms`), else return the error. `from_env(provider, model, api_url)` resolves base URL + auth + default model.
- [ ] Test the clients against **recorded response fixtures** (no live API): feed a canned Anthropic/OpenAI JSON body through the response-extraction + `parse_verdicts` path and assert the verdicts; test the retry decision (mock a 429 then 200 if feasible, or unit-test the backoff/decision function in isolation). Commit `Phase 4c: verdict cache + anthropic/openai(local) clients + retry`.

## Task 5: judge loop + report writer + summary (full flow)

**Files:** `crates/nameparser-cli/src/validate.rs` — the `run_validate` wire-up.

- [ ] **`run_validate`:** fail-fast if `--input` missing (`exit(2)`, helpful message). `select` (Task 2). Open the cache (Task 4). If not `--dry-run`, construct the `Judge` (Task 4) — a missing credential aborts here (not in dry-run). **Phase-2 loop:** iterate `chosen` in `--batch` chunks; per chunk split into cache-hits (resolved) vs `uncached`; if `uncached` non-empty and not dry-run, ONE `judge(user_message(uncached), uncached.len())` call; reconcile verdicts by echoed `index` (0-based within the uncached sub-batch; missing → `None`, never cached); `put` fresh verdicts to the cache; write one JSONL report row per chunk item (`line`, `input`, `parsed` or `error`, plus `verdict`/`confidence`/`note`/`fields` when present); per-chunk stderr progress. Close the cache in a `finally`-equivalent (Rust: ensure flush on all paths).
- [ ] **Summary** (stderr, non-dry-run): `Validated N names in C API call(s), F from cache.` / `ok=X suspect=Y wrong=Z (no verdict=M)` / top-15 most-flagged-`fields[].name` histogram (descending) / `Report → {path}` + a `jq` hint. Dry-run prints the Task-2/3 dry-run summary + first-batch payload instead.
- [ ] **End-to-end test:** `validate --input=<small corpus> --dry-run --cache=none --output=<tmp>` → reproducible report + summary, 0 API calls; assert the report rows + that a stale cache isn't consulted. Commit `Phase 4c: judge loop + report + summary — validate end to end (dry-run)`.
