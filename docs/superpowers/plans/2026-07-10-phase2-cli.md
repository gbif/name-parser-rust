# Phase 2 — Native CLI + cross-validation + full-parser benchmark

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use `- [ ]`.

**Goal:** Ship the native Rust CLI (`parse` / `benchmark` / `compare`, mirroring the Java CLI), measure the **full Rust parser throughput vs Java's 28 µs/name** (the comparison the Phase-0 spike deferred), and cross-validate Rust vs Java over the available corpora → declare parity.

**Architecture:** A new `nameparser-cli` workspace crate (clap) wrapping `nameparser::parse`, streaming stdin/file → JSONL (reusing the model's Gson-faithful serde, which is corpus-proven at 30/30). `benchmark` times the full pipeline. `compare` diffs two JSONL streams (the parity tool). Cross-validation = Rust CLI JSONL vs the Java CLI JSONL (`compare`), over `benchmark-data.txt` (8017, already 30/30 in-harness) + the Java test-resource corpora.

**Data note:** `col-names.tsv` (6.3M rows) is NOT present locally, so the "1M+ row" cross-validation is deferred — this slice cross-validates over the 8017 benchmark + the Java test corpora (`names-with-authors.txt`, `doubtful.txt`, `hybrids.txt`, `other.txt`, `otu.txt`, `placeholder.txt`, `viruses.txt` from `name-parser/src/test/resources/`). The full 6.3M run awaits the user dropping `col-names.tsv` into `testdata/`.

**Tech Stack:** Rust 2021, `clap` (new dep — the only one; CLI-crate only, not the core), reuse `nameparser` + `serde_json`. Java CLI shaded jar for the oracle side of `compare`.
